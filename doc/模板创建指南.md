# max-md2cv 简历模板创建规范说明书

本文档用于指导开发者或高级用户如何在本项目中创建和注册新的简历 CSS 模板。

## 一、模板文件位置及命名规则

1. **文件位置**：所有内置模板均放置在 `src/assets/templates/` 目录下。
2. **文件扩展名**：模板文件必须为标准的 `.css` 文件，例如 `my-template.css`。
3. **文件名作为 ID**：此 `.css` 文件的文件名（不包含扩展名）将作为模板在系统中的唯一 `id`（例如：`business`）。

## 二、模板关键元数据声明

每个模板文件的开头，**必须**使用特殊的 CSS 注释来声明模板的可视化名称。
该注释的格式如下（请置于文件第一行）：

```css
/* @name: 你的模板名称 */
```
例如：`/* @name: 极简极客 */` 或 `/* @name: 现代商务 */`。
后台加载模板列表时，会自动解析该字段用作应用内切换列表显示的名称。

## 三、预置的 CSS 变量 (注入变量)

为了让用户的 UI 配置项（如字体、颜色、字号等）在模板中无缝生效，前端渲染框架会在顶层 `.resume-document` 容器内注入以下 CSS 变量。**模板的样式编写应当尽可能使用这些变量来响应用户的定制，而不是硬编码死值**：

| CSS 变量名 | 用途 | 默认值示例 |
| --- | --- | --- |
| `--cv-theme-color` | 简历主题色（常用于标题颜色、图标颜色、边框、底色修饰等） | `#4c49cc` |
| `--cv-font-size` | 正文基础字号 | `14px` |
| `--cv-paragraph-spacing` | 段落间距 | `8px` |
| `--cv-h1-size` | 简历主名称(H1)字号 | `32px` |
| `--cv-h2-size` | 二级标题(模块标题如“工作经验”)字号 | `18px` |
| `--cv-h3-size` | 三级标题(公司、职位等)字号 | `16px` |
| `--cv-date-size` | 右侧经验时间的字号 | (基于比例缩放) |
| `--cv-date-weight` | 右侧经验时间的字重 | `bold` |
| `--cv-photo-width` | 证件照容器宽度 | `90px` |
| `--cv-photo-height`| 证件照容器高度 | `120px` |
| `--cv-photo-reserve`| 预留给证件照的避让宽度(自带安全间距) | `calc(width + gap)` |

*(注: `.resume-document` 本身的 `font-family`、`line-height` 会自动生效并作用到所有子节点，通常不需要模板再写覆盖逻辑，除非特定元素需要定制字号。)*

## 四、核心的 DOM 层级结构与可控类名

简历 Markdown 最终会被解析和生成一系列标准的 HTML 层级，所有内容均包裹在顶层 `<div class="resume-document"></div>` 内部：

### 1. 基础段落标签
*   `h1` ~ `h4`：逐级标题。
*   `p`：普通正文。
*   `ul` / `ol` / `li`：项目列表描述。
*   `blockquote`：引用文字，通常在写自我评价时使用。

### 2. 横断求职意向块 (`.job-intention`)
如果在 Markdown 首部存在包含 "求职意向"、"期望职位" 等关键词的段落，程序会自动将其挂载 `job-intention` 类名并置顶颜色：
```html
<p class="job-intention" style="color: var(--cv-theme-color);">求职意向: 前端工程师...</p>
```

### 3. 联系方式解析块 (图标模式与文本模式)
解析器会自动将紧跟意向的个人基础信息段落（如性别、年龄、电话、邮箱、Github）解析转化为统一结构。
**纯文本模式 (`.contact-info--text`)**：
```html
<div class="contact-info contact-info--text">
  <p class="contact-info-text-line">邮箱：xxx@xxx.com | 电话：123...</p>
</div>
```
**自动图标模式 (`.contact-info--icon`)**：
```html
<div class="contact-info contact-info--icon">
  <span class="contact-info-item contact-info-item--email">
     <span class="contact-info-icon"><svg>...</svg></span> <!-- 图标默认继承字体颜色，可自行加色 -->
     <span class="contact-info-value">xxx@xxx.com</span>
  </span>
  ...
</div>
```
***特别注意**：是否显示图标化联系方式，取决于模板 CSS 中是否声明了 `--cv-contact-render: icon;`。程序会自动从 CSS 变量中解析该设置，**无需手动修改任何 TypeScript 代码**。*

### 4. 经验记录栏拆解 (`.experience-line`)
对于具有时间段特征分布的行（如以 `[2021.05 - 2023.08]` 结尾），解析器会自动提取时间并将其分为两端结构，以简化不同分辨率下的左右对齐：
```html
<h3 class="experience-line">
  <span class="experience-title">某公司 - 高级工程师</span>
  <span class="experience-date">2021.05 - 2023.08</span>
</h3>
```

### 5. 证件照占位容器避让 (`.resume-photo-wrapper`)
无论是否上传，如果用户在右下角配置了“照片”，DOM 树第一顺位会注入证件照包装盒并绝对定位在右上方（`position: absolute; right: 0; top: 0;`）。
由于是脱离文档流的绝对定位，所以模板作者 **必须在顶部的元素（如 `h1`、`.job-intention` 或 `.contact-info`）中增加 `padding-right: var(--cv-photo-reserve)`** 来为照片预留空间，以防页面缩小时文字穿模。

## 五、如何发布/打包新模板？

**本项目模板系统的最大特点在于“全自动注册加载”。**

当你在 `src/assets/templates/` 中放好带有 `/* @name: xxx */` 首段注释的 `.css` 文件后：
1. **在开发模式中（dev）**：由于采用资源目录按需打包，直接新增或修改可能不会被热更新立即拾取到。您只需要在终端重启 `npm run tauri dev` 命令即可生效。
2. **构建打包（build）**：`tauri.conf.json` 内部的 bundle resources 中已经将 `../src/assets/templates/*.css` 配置为资源打包项，Rust 终端利用 `list_templates` 函数全自动搜寻加载此目录下的全部样式，并通过 Tauri RPC 同步推送给前端 UI 供用户选择。

—— **您不需要向任何前端字典配置、Vue路由或 store 数组中人工硬编码注册新模板的 ID！**
