# max-md2cv 模板创建指南

本文档描述当前项目已经落地的模板系统用法。  
如果你要新增一个内置模板，或者准备做用户模板包，请以本文档为准，不要再按照旧的 `src/assets/templates/*.css` 或 `*.manifest.json` 方式创建。

## 1. 当前模板系统是什么

现在的模板系统已经从“切换一份 CSS 文件”升级为“模板包”：

- 每个模板是一个独立目录
- 模板目录至少包含 `template.json` 和 `style.css`
- 模板默认值、schema 预设声明、少量字段覆写、功能声明写在 `template.json`
- 模板视觉样式写在 `style.css`
- 桌面端和 Web 端共用同一套内置模板源

当前内置模板目录位置：

```text
packages/resume-core/src/assets/templates/
  modern/
    template.json
    style.css
  classic/
    template.json
    style.css
  business/
    template.json
    style.css
```

## 2. 模板放在哪里

### 2.1 内置模板

内置模板统一放在：

[`packages/resume-core/src/assets/templates`](/d:/Project/max-md2cv/packages/resume-core/src/assets/templates)

不要再往根目录的 `src/assets/templates` 放模板资源，那套旧路径已经不再是模板单一来源。

### 2.2 用户模板

桌面端运行后，用户模板会保存在应用数据目录下的：

```text
AppData/templates/<template-id>/
  template.json
  style.css
```

Rust 侧会优先读取用户模板；如果用户模板和内置模板同 `id`，用户模板覆盖内置模板。

## 3. 模板目录结构

标准模板目录结构如下：

```text
my-template/
  template.json
  style.css
  assets/
```

其中：

- `template.json`：模板元信息、默认值、编辑字段 schema、功能声明
- `style.css`：模板主样式入口
- `assets/`：可选。放图片、SVG 等模板资源

当前代码已经支持目录模板。  
旧的单文件 `.css` 模板仍然能被兼容读取，但那只是过渡能力，不建议再新增旧格式模板。

## 4. template.json 结构

最小可用模板示例：

```json
{
  "id": "my-template",
  "name": "我的模板",
  "version": "1.0.0",
  "entryCss": "style.css",
  "description": "模板描述",
  "schemaPreset": "standard",
  "defaults": {
    "themeColor": "#4c49cc",
    "fontFamily": "\"Microsoft YaHei\", \"PingFang SC\", sans-serif",
    "fontSize": 14,
    "paragraphSpacing": 8,
    "h1Size": 32,
    "h2Size": 18,
    "h3Size": 16,
    "dateSize": 14,
    "dateWeight": "700",
    "lineHeight": 1.5,
    "marginV": 10,
    "marginH": 12,
    "personalInfoMode": "icon",
    "headerLayout": "stack",
    "photoPlacement": "top-right",
    "sectionTitlePreset": "accent-bar"
  },
  "features": {
    "photo": true,
    "pageFooter": true,
    "contactModes": ["text", "icon"]
  },
  "layout": {
    "headerLayout": "stack",
    "personalInfoMode": "icon",
    "photoPlacement": "top-right",
    "sectionTitlePreset": "accent-bar"
  }
}
```

### 4.1 必填字段

- `id`
- `name`
- `version`
- `entryCss`
- `defaults`
- `schemaPreset` 或 `editorSchema`

### 4.2 schemaPreset 推荐用法

当前内置模板推荐优先使用：

```json
{
  "schemaPreset": "standard"
}
```

这样模板会自动使用共享的标准字段 schema，不需要每个模板重复复制一整份 `editorSchema`。

如果模板只想对少量字段做调整，可以额外声明：

```json
{
  "schemaPreset": "standard",
  "editorSchemaOverrides": [
    {
      "key": "dateWeight",
      "label": "日期字重",
      "group": "字号"
    }
  ]
}
```

只有当模板真的需要完全脱离标准字段集合时，才建议直接写完整 `editorSchema`。

### 4.3 当前支持的字段类型

`editorSchema.type` 当前支持：

- `color`
- `number`
- `select`
- `boolean`
- `text`

### 4.4 当前已经在模板里使用的常见 key

- `themeColor`
- `fontFamily`
- `fontSize`
- `h1Size`
- `h2Size`
- `h3Size`
- `dateSize`
- `dateWeight`
- `lineHeight`
- `paragraphSpacing`
- `h2MarginTop`
- `h2MarginBottom`
- `h3MarginTop`
- `h3MarginBottom`
- `personalHeaderSpacing`
- `marginV`
- `marginH`
- `headerLayout`
- `personalInfoMode`
- `photoPlacement`
- `sectionTitlePreset`

## 5. style.css 应该怎么写

### 5.1 用模板变量，不要硬编码

运行时会把模板值注入到 `.resume-document` 上。模板 CSS 应优先消费这些变量，而不是把颜色、字号、页边距全部写死。

当前应优先使用的变量：

| 变量 | 说明 |
| --- | --- |
| `--tpl-theme-color` | 主题色 |
| `--tpl-font-family` | 字体族 |
| `--tpl-font-size` | 正文字号 |
| `--tpl-line-height` | 行高 |
| `--tpl-paragraph-spacing` | 段间距 |
| `--tpl-h1-size` | H1 字号 |
| `--tpl-h2-size` | H2 字号 |
| `--tpl-h3-size` | H3 字号 |
| `--tpl-h2-margin-top` | H2 上边距 |
| `--tpl-h2-margin-bottom` | H2 下边距 |
| `--tpl-h3-margin-top` | H3 上边距 |
| `--tpl-h3-margin-bottom` | H3 下边距 |
| `--tpl-date-size` | 日期字号 |
| `--tpl-date-weight` | 日期字重 |
| `--tpl-personal-header-spacing` | 抬头信息间距 |
| `--tpl-page-margin-v` | 上下页边距 |
| `--tpl-page-margin-h` | 左右页边距 |

当前运行时还保留了一组 `--cv-*` 兼容别名，但新模板不要再以它们为主。

### 5.2 示例

```css
@page {
  margin: var(--tpl-page-margin-v) var(--tpl-page-margin-h);
}

.resume-document {
  font-family: var(--tpl-font-family);
  font-size: var(--tpl-font-size);
  line-height: var(--tpl-line-height);
  color: #000;
}

.resume-document h2 {
  font-size: var(--tpl-h2-size);
  color: var(--tpl-theme-color);
  margin-top: var(--tpl-h2-margin-top);
  margin-bottom: var(--tpl-h2-margin-bottom);
}
```

## 6. 当前稳定的语义结构和钩子

Markdown 最终会被渲染为一套稳定的语义化 DOM，模板 CSS 应该围绕这些结构写，而不是依赖偶然的标签位置。

常用结构：

- `.resume-document`
- `.resume-header`
- `.resume-header-body`
- `.resume-header-main`
- `.resume-header-meta`
- `.resume-header-photo`
- `.resume-photo-wrapper`
- `.personal-header`
- `.job-intention`
- `.contact-info--text`
- `.contact-info--icon`
- `.contact-info-item`
- `.experience-line`
- `.experience-title`
- `.experience-date`
- `.manual-page-break`

### 6.1 运行时 data 属性

预览和导出时，根节点上还会带这些属性：

- `data-header-layout`
- `data-personal-info-mode`
- `data-photo-placement`
- `data-section-title-preset`

因此模板可以这样写布局差异：

```css
.resume-document[data-photo-placement="header-right"] .resume-header {
  display: flex;
  justify-content: space-between;
}
```

## 7. 联系方式和照片

### 7.1 联系方式

联系方式渲染有两种语义模式：

- `.contact-info--text`
- `.contact-info--icon`

模板可以通过：

- `defaults.personalInfoMode`
- `layout.personalInfoMode`

控制默认渲染方式。

### 7.2 照片

照片相关的公共能力由运行时提供，模板主要消费以下结构和变量：

- `.resume-photo-wrapper`
- `--cv-photo-width`
- `--cv-photo-height`
- `--cv-photo-gap`
- `--cv-photo-reserve`

如果模板使用右上角照片布局，需要为抬头文字预留避让空间；如果使用 `header-right`，则可直接依赖 `.resume-header` 的结构化布局。

## 8. editorSchema 和当前 UI 的关系

当前模板系统已经是 schema-driven，但预览页并不会把所有 schema 字段都暴露给用户。

目前预览页顶部工具栏：

- 会根据 `editorSchema` 自动渲染基础、字号、间距等字段
- `dateWeight` 在预览页里会用分段控件呈现
- 不允许在预览页直接调整布局字段

当前在预览页被隐藏的布局字段：

- `headerLayout`
- `personalInfoMode`
- `photoPlacement`
- `sectionTitlePreset`

这几个字段依然可以写在模板里，也会参与默认值和渲染，只是现阶段不从预览工具栏开放。

## 9. 新增模板后需要改代码吗

正常情况下不需要。

如果你新增的是一个合规模板目录：

1. 新建模板目录
2. 放入 `template.json`
3. 放入 `style.css`
4. 如有资源，再放 `assets/`

桌面端和 Web 端都会自动读取。

原因是：

- Web 内置模板通过 [loader.ts](/d:/Project/max-md2cv/packages/resume-core/src/templates/loader.ts) 从共享目录导入
- 桌面端通过 [templates.rs](/d:/Project/max-md2cv/src-tauri/src/templates.rs) 扫描模板包目录
- Tauri 打包通过 [tauri.conf.json](/d:/Project/max-md2cv/src-tauri/tauri.conf.json) 把共享模板目录整体带入资源

## 10. 开发和构建时如何生效

### 10.1 Web

如果你改的是内置模板目录，重新执行 Web 构建或开发启动即可。

### 10.2 桌面端

桌面端开发模式会优先读取仓库里的共享模板目录；构建后则读取打包进资源目录的模板。

因此：

- 开发时，修改 `packages/resume-core/src/assets/templates` 后重启开发进程即可
- 构建时，模板目录会整体随应用一起打包

## 11. 当前后端命令

模板相关命令当前是：

- `list_templates`
- `save_template_package`

其中：

- `list_templates` 负责读取内置模板和用户模板
- `save_template_package` 负责把完整模板包写入用户模板目录

## 12. 不建议再做的旧写法

以下写法已经过时，不要继续新增：

- 在 `src/assets/templates` 下放模板
- 只创建单个 `.css` 文件作为新模板
- 依赖 `*.manifest.json`
- 依赖 `/* @name: xxx */` 注释作为唯一元信息来源
- 主要依赖 `--cv-*` 变量写新模板
- 假设预览工具栏里会自动开放布局字段

## 13. 推荐起步方式

如果你要新建模板，最稳妥的方式是：

1. 复制一个现有模板目录，例如 `modern/`
2. 修改 `template.json` 的 `id`、`name`、`defaults`
3. 如有必要，再补 `editorSchemaOverrides` 或完整 `editorSchema`
4. 修改 `style.css`
5. 启动应用检查预览和导出效果
