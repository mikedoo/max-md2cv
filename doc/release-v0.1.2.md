# v0.1.2 Release Notes

发布日期：2026-04-13

## 概览

`v0.1.2` 聚焦三件事：补齐 Web Playground、把模板系统整理成可扩展的清单化结构，以及提升预览到导出的链路一致性。

这一版同时完成了桌面端状态拆分与共享核心沉淀。模板、样式和简历相关类型被统一收敛到 `packages/resume-core`，为桌面端与 Web 端共用能力打下了稳定基础。

## 主要更新

### 1. Web Playground 正式补齐基础能力

- 新增 `apps/web` Web 应用入口，支持在浏览器中编辑、预览和调试简历。
- Web 端复用桌面端核心组件与共享模板逻辑，减少双端分叉。
- 修复了 Web 预览异常并同步调整了侧边栏与页面布局表现。

### 2. 模板系统重构为 Manifest + Schema 驱动

- 内置模板资源迁移到 `packages/resume-core`，桌面端与 Web 端统一从共享包加载。
- 为模板新增 `template.json` 清单文件，模板元信息、样式文件和能力边界更清晰。
- 模板样式控制改为基于 schema 的预设与解析流程，后续新增模板和样式项更容易维护。
- 清理旧模板路径和废弃资源，减少历史兼容负担。

### 3. 预览与导出链路进一步对齐

- 持续沿用浏览器打印导出路径，同时增强运行时样式生成和分页导出逻辑。
- PDF 导出相关状态与流程被拆分到更清晰的 store 模块，便于维护和扩展。
- 改善预览工具栏与模板渲染控制，减少预览和最终 PDF 之间的样式偏差。

### 4. 照片与版式调节体验增强

- 增强头像照片的调整与渲染逻辑，提升模板中的照片展示稳定性。
- 优化无照片场景下的占位 UI，空状态反馈更明确。
- 新增并修复多个版式参数，包括 `H2`、`H3` 与个人信息头部间距控制。

## 其他改进

- 将桌面端 `resume` store 继续拆分为更明确的职责模块。
- 更新版本号到 `0.1.2`，同步桌面端 Tauri 配置与 Rust 包版本。
- 更新 README、项目说明和开发文档，使目录结构与当前实现保持一致。

## 升级说明

- 桌面端文件读写仍通过 Tauri 命令完成，未引入新的前端直连文件系统路径。
- 模板加载现在依赖共享清单结构；自定义模板若基于旧目录结构实现，建议按新模板资源组织方式校验一次。
