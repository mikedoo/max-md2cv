# v0.1.2 Release Notes

发布日期：2026-04-02

## 概览

`v0.1.2` 是一次以架构整理、Web Playground 引入和导出体验提升为核心的版本更新。

本次版本将简历解析、模板样式和字体资源沉淀到共享包 `@resume-core`，同时将桌面端简历状态拆分为模块化 Pinia 结构，降低后续功能扩展成本，并为 Web 端 Playground 提供统一基础能力。

## 主要更新

### 1. 新增 Web Playground

- 新增独立的 `apps/web` Web 应用入口。
- 支持在浏览器中进行简历预览与样式调试。
- 复用桌面端核心解析与样式能力，减少双端逻辑分叉。

### 2. 提取共享核心包 `@resume-core`

- 新增共享的模板、样式主题、字体资源与简历类型定义。
- 集中管理 Markdown 渲染、分页标记、模板样式解析等核心逻辑。
- 为桌面端与 Web 端提供统一的业务基础，减少重复实现。

### 3. 重构桌面端状态管理

- 原有较大的 `resume` store 已拆分为多个职责明确的模块。
- 新增工作区文件、导出、渲染配置、照片处理、监听等子模块。
- 状态职责更清晰，后续维护与测试更容易落地。

### 4. 提升预览与导出一致性

- 新增运行时样式构建与分页导出文档生成逻辑。
- 优化 PDF 导出链路中的样式注入与页面边距控制。
- 改善预览表现与最终导出结果之间的一致性。

### 5. 增强样式自定义能力

- 新增对 `H2`、`H3` 和个人信息头部间距的可配置支持。
- 模板样式解析能力增强，可从模板 CSS 中恢复更多样式参数。
- 为后续模板扩展和个性化排版留出更稳定的接口。

## 其他变更

- 应用版本号更新至 `0.1.2`。
- 调整部分界面组件与主题样式，统一双端视觉基础。
- 更新相关文档与工程配置以适配新的目录结构。

## 升级影响

- 桌面端仍保持当前 Tauri 浏览器打印导出路径，不引入额外导出通道。
- 由于核心逻辑已向 `@resume-core` 聚合，今后双端能力会更容易保持一致。
